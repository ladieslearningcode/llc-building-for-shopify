What is "hard coding"?

Let's take a step back and learn a bit about HTML.

WHAT IS HTML?

Hypertext Markup Language

HTML

HTML is a markup language. It is the essential language of the web. It defines the content of a web page.

Just like languages that we speak and write, HTML follows proper "grammar" rules, too. HTML tags are written with left & right angled brackets < > and the forward slash /. Get to know these keys!

What is "hard coding"?

When text is explictly written in the HTML, we say that it is hard coded. This means it can only be changed if someone has access to the site's files, and knows how to code.

HTML


<h1>Fuzzy boots</h1>
<h3>$139.99</h3>
<p>These boots will keep you warm all winter. Their truly amazing!</p>

How does Liquid help?

Instead of writing out explicity what we want, we can reference a liquid object.

Liquid objects will dynamically display the value of an attribute, like a title, price, etc.
within a set of handles: {{ ... }}

Liquid


<h1>{{ product.title }}</h1>
<p>{{ product.description }}</p>

HTML returned


<h1>Rooster Decor Wood Cheese Board</h1>
<p>This is such a lovely wood grain rooster cheese board.
Great country / rustic decor for the rooster collector.</p>

Exercise #6c: Show hero images only if they exist

We can use Control Flow Tags to check if a hero image exists.

Liquid tags are used for programming logic. Tags are wrapped with: {% ... %}

Pseudo code


{% if collection_image_exists %}
    Show the hero image
{% endif %}

Pro tip: Pseudo code means we're using simple language to think out the steps of the problem. It's intended for human reading, so we're not worrying about the proper syntax. Think of it as a rough draft that we can polish later!

if-statements

Only does something if a condition is true


{% if a condition is true %}
    do this...
{% endif %}

Have different results when different conditions are met


{% if primary condition is true %}
    Do this.
{% elsif secondary condition is true %}
    Do this.
{% else %}
    When all else fails, do this.
{% endif %}

Show a discount code

How would you show a discount code on a Product page if the product is part of a specific collection - in this case the Household collection?

  • Make a discount code that will only work for products in the Household collection.
  • Modify product-template.liquid to display the code in an if-statement.

Make a discount code

  • Code: home4holidays
  • Options: 20% off
  • Applies to: Products in the the Household collection.
  • Optional: You can set an end date to expire this discount after the holidays are over

Iteration Tags

Now we want to display this discount code on any product that is part of this collection. Since a product can be part of multiple collections, we to check each collection this product is a part of.

We can use an Iteration Tag to loop through the collections:

Pseudo code


{% for each collection in collections %}

    {% if title is 'Household' %}
      Show the discount code
    {% endif %}

{% endfor %}

Introducing the For-loop

A loop is when an instruction is repeated over and over again until a task is completed. Think about a sink full of glasses that you need to wash - you will repeat the same actions with each glass until there are no glasses left in the sink. That's a loop!

Introducing the For-loop

Let's build a for loop based on our pseudo code.

What product property can we use to get information on a product's collections?

Answer: {{ product.collections }}

Introducing the For-loop

What collection property tells us the title?

Answer: {{ collection.title }}

What operator checks if two things are equal?

Answer: ==


{% for collection in product.collections %}
  {% if collection.title == 'Household' %}
    Show the discount code.
  {% endif %}
{% endfor %}

Theme tags

This file contains a piece of code, {{ content_for_index }}, that injects home page sections.

Elsewhere in theme files you may also see {% include %} elements in the template. These are a type of Theme tag.

We use include to pull parts of other files into templates.

collection-template.liquid


{% if paginate.pages > 1 %}  // if we have enough products to merit pages
  {% include 'pagination' %} // show the pagination buttons
{% endif %}

Theme Tags

Let's see if we can determine where the home page collections are coded!

Open up the Section titled collection.liquid and find where the code is displaying products.

                  
<div class="grid grid--uniform grid--view-items">
  {% for product in collection.products %}
    <div class="grid__item {{ grid_item_width }}">
      {% include 'product-card-grid', grid_image_width: image_size %}
    </div>
  {% endfor %}
</div>

Where you see the word include, the contents of another file are being pulled into this one.
The product-card-grid.liquid snippet will include all the code to display a product's image, title, price, vendor, etc.

Parameters

Now we have to split the products up into separate pages. Is there a for-loop parameter that can help?

                  
<div class="grid grid--uniform grid--view-items">
  {% for product in collection.products limit: 12 %}
    <div class="grid__item {{ grid_item_width }}">
      {% include 'product-card-grid', grid_image_width: image_size %}
    </div>
  {% endfor %}
</div>

Add this limit to your collection-template.liquid and check out one of your collection pages to see the changes!

Bogus Gateway

Test the order process

Bogus Gateway

You may want to simulate a transaction on your store for a number of reasons:

  • Customizing order confirmation emails
  • Trying out cart modifications
  • To see the order status page

The best way to do that in a development shop is by activating the Bogus Gateway.

Your development shop will only allow you to make 10 "real" transactions, but you can make unlimited transactions through the Bogus Gateway.

Note: The Bogus Gateway is only available to development stores under a Partners account.

Buying something from yourself, and then canceling it or refunding counts as a "real" transaction. Buying something as "Cash on delivery" also counts as a "real" transation.
